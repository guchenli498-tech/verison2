## 3. 运行环境要求

### 3.1 软件依赖

- Node.js：建议 LTS（18+ 或 20+）
- npm：与 Node.js 配套
- Git：可选（用于版本管理）
- 浏览器：Chrome / Edge

### 3.2 机器配置建议

- CPU：双核及以上
- 内存：8GB 及以上（推荐 16GB）
- 磁盘：至少 2GB 空闲空间

### 3.3 系统兼容性

- Windows 10/11
- macOS（Intel/Apple Silicon）
- Linux（主流发行版）

---

## 4. 目录约定

以下路径以项目根目录为基准：

- `frontend/`：主工程目录（命令执行目录）
- `frontend/src/`：React 业务源码
- `frontend/public/charts/`：模块二、三图表页和资源
- `frontend/dist/`：生产构建输出目录（构建后生成）

---

## 5. 首次部署流程

### 5.1 进入工程目录

```bash
cd frontend
```

### 5.2 安装依赖

```bash
npm install
```

### 5.3 启动开发服务

```bash
npm run dev
```

### 5.4 验证启动结果

启动成功后终端会输出地址，例如：
- `http://localhost:5173`

在浏览器访问后，需确认：
- 首页可打开
- 可进入模块一、模块二、模块三
- 页面无明显空白或报错

## 6. 日常运行流程（开发/演示）

### 6.1 开发模式

```bash
cd frontend
npm run dev
```

说明：
- 支持热更新
- 适合调试页面样式、图表配置和交互逻辑

### 6.2 演示模式（建议）

```bash
cd frontend
npm run build
npm run preview
```

说明：
- 使用生产构建结果进行演示更稳定
- 有助于提前发现构建期问题

## 7. 构建与发布流程

## 7.1 生产构建

```bash
cd frontend
npm run build
```

执行内容：
- TypeScript 构建检查（`tsc -b`）
- Vite 生产构建

输出目录：
- `frontend/dist`

### 7.2 本地预览

```bash
npm run preview
```

用于验证构建产物在接近生产环境下的表现。

### 7.3 静态部署建议

将 `dist` 完整部署到静态服务器（Nginx / 静态托管平台），并配置 SPA 回退规则：
- 当路径不存在真实文件时，回退到 `index.html`

## 9. 常见故障与排障流程

## 9.1 依赖安装失败

### 现象

- `npm install` 中断
- 报网络超时或权限错误

### 处理步骤

1. 检查 Node.js/npm 版本是否过旧
2. 删除 `node_modules` 与 `package-lock.json` 后重装
3. 切换稳定网络或镜像源后重试

## 9.2 端口占用

### 现象

- 启动时提示端口冲突

### 处理步骤

1. 结束占用端口进程，或  
2. 指定新端口启动：

```bash
npm run dev -- --port 5174
```

## 9.3 页面空白 / 资源 404

### 重点检查

1. 是否在 `frontend` 目录启动
2. `public/charts` 文件是否缺失
3. 浏览器控制台是否有路径错误

## 9.4 构建失败

### 可能原因

- TypeScript 类型错误
- ESLint 语法/规范错误
- 文件路径大小写不一致

### 处理步骤

1. 先执行 `npm run lint`
2. 根据报错修复对应文件
3. 再执行 `npm run build` 验证

## 9.5 本地可用但预览异常

### 可能原因

- 开发模式依赖 HMR 特性，生产模式更严格

### 处理步骤

1. 使用 `npm run preview` 复现
2. 检查生产环境资源路径是否正确
3. 检查是否依赖了仅开发环境可用的行为

## 10. 运行巡检清单（每次演示前）

### 10.1 基础巡检

- [ ] 终端无启动报错
- [ ] 首页加载正常
- [ ] 模块路由可点击进入

### 10.2 内容巡检

- [ ] 模块一地图和折线图可见

### 10.3 构建巡检

- [ ] `npm run lint` 通过
- [ ] `npm run build` 通过
- [ ] `npm run preview` 可访问

## 11. 交付与发布前验收

建议按以下标准完成最终验收：

1. **功能完整性**：首页与三大模块全部可达  
2. **交互可用性**：关键按钮、筛选、图例、提示均可用  
3. **视觉一致性**：配色、字体、标题规范一致  
4. **运行稳定性**：连续运行 30 分钟无关键故障  
5. **可复现性**：新环境按手册可在 30 分钟内拉起

## 12. 运行维护建议

### 12.1 版本管理

- 每次发布记录版本号与变更点

- 保留文档与资源的离线备份
